round() 函数的副作用

当你使用 round(x, 3) 时，Python 会尝试将这个“无限逼近的二进制数”四舍五入到小数点后 3 位，并生成一个新的浮点数。
由于上述的精度问题，这个新生成的浮点数在底层依然无法完美表示 0.123456。当你用 print() 直接打印这个经过 round() 处理的数字时，Python 可能会输出类似 0.12345600000000001 或 0.12345599999999999 这样的结果，这就导致输出与输入不完全一致。
在算法题目中，永远优先使用字符串处理或格式化输出。

Input

在 Python 中，当输入的数据在同一行且以空格分隔时（如 1 2），最常用且最优雅的处理方式是使用 split() 方法。

split() 方法

split() 是 Python 字符串（String）对象的一个非常核心的内置方法。它的核心作用是按照指定的分隔符，将一个完整的字符串“切”成多段，并返回一个包含这些段的列表（List）。
基本语法

  • sep（分隔符）：指定用什么字符来切割。如果不写，默认以所有连续的空白字符（空格、换行符 \n、制表符 \t 等）作为分隔符。也可以按行切分。
  • maxsplit（最大切割次数）：指定最多切几刀。如果不写，默认 -1，表示切到底，有多少分隔符就切多少次

map()方法

基本语法

  • function：要应用的函数。可以是内置函数（如 int）、自定义函数或 lambda 匿名函数。
  • iterable：一个或多个可迭代对象（如列表、元组、字符串等）。
    map() 是 Python 中的一个内置高阶函数。它的核心作用是将一个指定的函数依次应用于一个或多个可迭代对象（如列表、元组等）的每个元素，并返回一个包含处理结果的新迭代器。
    map() 函数返回的是一个迭代器（map object），而不是一个列表。它采用“惰性计算”策略，即不会立即对所有元素进行计算，只有在真正需要用到结果时才会逐个计算。
    注意：如果你想直接查看或打印结果，必须使用 list() 或 tuple() 将其转换。例如：list(map(...))。
    应用场景：
  • 场景一：对单个列表进行数据转换
  • 场景二：结合 lambda 函数进行数学运算
  • 场景三：处理多个可迭代对象
  • 场景四：处理嵌套数据结构
