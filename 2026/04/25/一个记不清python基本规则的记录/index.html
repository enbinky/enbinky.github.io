我们需要什么样的虚拟环境？

Python 原生虚拟环境（venv/virtualenv）

Python 官方内置的轻量级 Python 专属隔离工具，仅解决”不同 Python 项目之间的 pip 包冲突“问题。只适用于纯python环境。最大的优点是体积小，轻量化。

绝对不适用场景：需要安装 CUDA、cuDNN、MKL、FFmpeg 等系统级二进制依赖；需要切换不同 Python 大版本；依赖包含 C/C++ 扩展且编译复杂（如 numpy、scipy 旧版本）；多语言混合开发（Python + R/Julia/C++）。

笔者自述：笔者使用了数周的（venv）Python原生环境，遇到数不清的包冲突问题。在复杂科研和机器学习等环境下，绝不推荐！！！

所以，推荐使用conda虚拟环境。

插入：使用conda环境前，需要先卸载venv虚拟环境。
