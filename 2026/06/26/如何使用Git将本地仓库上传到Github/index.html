主要分两步：Github创建远程仓库+Git指令上传本地仓库

Github创建远程仓库

在Github上创建远程仓库，并记录代码仓库链接（以”Web_Link”为例）

Git指令上传本地代码

首先，在代码本地根目录下打开Git Bash

Git限制

Git本地代码仓库的大小与项目大小无关，主要与文件类型有关。
对于纯文本代码项目，可以放心大胆的提交。
对于大型资源文件，建议从一开始就配置 Git LFS（大文件存储），或者云盘配置链接。都有一些免费额度，但是多了还是要付费。

Git Large File Storage

Git LFS（Git Large File Storage）是由 GitHub 开发的一款 Git 扩展工具，旨在帮助开发者更高效地管理仓库中的大文件。传统 Git 会将文件的每个版本完整存储在仓库历史中，导致大文件（如音频、视频、数据集、二进制文件等）快速膨胀仓库体积，影响克隆和推送效率。
git lfs 只适用于大型二进制文件、仓库体积过大、机器学习模型与大型训练数据等情况。
